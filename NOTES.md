# Best Configuration and Why it Works

Our best configuration replaces the baseline byte-level tokenizer with a custom hybrid character-byte tokenizer that individually tokenizes the unique multi-byte UTF-8 sequences (such as Devanagari characters) found in the training corpus. This reduces the number of tokens required to encode Devanagari script from 3 tokens per character to just 1 token, substantially increasing the effective capacity of the model's context window. To stay within the 2,000,000 parameter budget, we tied the input embedding and output projection head weights, which allowed us to increase the model dimension (`n_embd`) from 160 to 192 and the number of attention heads (`n_head`) from 4 to 6. Furthermore, replacing the constant learning rate and basic Adam optimizer with AdamW, weight decay, gradient clipping, and a cosine decay learning rate scheduler with warmup significantly accelerated convergence within the strict 2,000-step limit. Together, these modifications achieved a Bits Per Byte (BPB) of 2.1801, a massive improvement over the 2.3718 baseline.
